# Return-to-normality-From-cam-distortion
This is a tool that performs camera calibration and corrects lens distortion using OpenCV. 


# Return-to-Normality-Cam-Distortion

본 프로젝트는 OpenCV 라이브러리를 활용하여 카메라를 캘리브레이션(Calibration)하고, 렌즈의 기하학적 왜곡(Geometric Distortion)을 수학적으로 계산하여 평평하고 정상적인 이미지로 보정(Undistort)하는 Python 스크립트입니다.

## ⚙️ 구현 파라미터 튜닝 
카메라의 내부 파라미터를 정확하게 추출하고 연산 효율을 높이기 위해 다음과 같이 파라미터를 설정했습니다.

1. **체스보드 규격 매핑 (`PATTERN_SIZE`, `SQUARE_SIZE`):** 사용한 체스보드 패턴에 완벽히 대응하도록 내부 십자 코너 개수를 `(8, 6)`으로 설정하고, 실제 한 칸의 물리적 크기를 `0.032m` (32mm)로 정밀하게 입력하여 3D 공간상의 좌표 스케일을 맞췄습니다.
2. **프레임 샘플링 최적화:** 동영상의 모든 프레임을 연산할 경우 발생하는 부하를 줄이기 위해 `frame_count % 5 != 0` 조건을 부여했습니다. 매 5프레임마다 체스보드 코너를 탐색하도록 하여 연산 속도를 높이면서도 캘리브레이션에 필요한 다양한 각도의 데이터(Coverage)를 충분히 확보했습니다.
3. **서브픽셀 정밀도 보정 (`cv2.cornerSubPix`):** 코너의 위치를 픽셀 단위보다 더 정밀한 소수점 단위로 찾기 위해 반복 연산(최대 30회, 오차 0.001) 기준을 적용하여 캘리브레이션의 오차율을 최소화했습니다.

## 🎬 캘리브레이션 및 보정 결과 데모

극단적인 렌즈 왜곡 보정 전후를 시각적으로 확인하기 위해, 인위적인 초광각 이펙트(왜곡)가 적용된 체스보드 동영상을 촬영하여 알고리즘을 적용했습니다.

### 1. 캘리브레이션 산출 파라미터
* **RMS Error (재투영 오차):** `1.0368`
* **Camera Matrix (내부 파라미터 K):**
  * $f_x$: `473.7452` / $f_y$: `488.8001`
  * $c_x$: `628.5978` / $c_y$: `362.1670`
* **Distortion Coefficients (왜곡 계수):**
  * `[ 0.20936315 -0.22016411  0.00337147 -0.00748419  0.10756746]`

### 2. 왜곡 보정 결과 (Original vs Undistorted)
좌측의 왜곡된 원본 영상(`Original`)과 우측의 기하학적 왜곡이 펴진 보정 영상(`Undistorted`)을 나란히 비교한 데모입니다.

## 🔍 한계점 및 논리적 분석 

### 1. 스마트폰 ISP의 자동 보정 개입 한계
최초 촬영 시 스마트폰의 일반 렌즈 및 광각 렌즈를 사용하여 동영상을 촬영했으나, 최신 기기 내부의 이미지 프로세서(ISP)가 영상을 저장하는 단계에서 이미 볼록 왜곡(Barrel Distortion)을 자체적으로 펴버리는 현상이 발생했습니다. 이로 인해 초기 캘리브레이션 결과에서 왜곡 계수($k_1$)가 0에 수렴하여 보정 전후의 차이를 극적으로 보여주기 어려운 하드웨어/소프트웨어적 한계가 있었습니다.

### 2. 전략 수정 및 한계 극복 (오목 왜곡 역보정)
위 한계를 극복하고 컴퓨터 비전 알고리즘의 강력함을 증명하기 위해, 소프트웨어적인 '초광각 이펙트'를 영상에 인위적으로 부여하는 전략을 취했습니다. 
그 결과, 알고리즘이 화면 가장자리가 안쪽으로 빨려 들어가는 **오목 왜곡(Pincushion Distortion, $k_1 \approx 0.21$)** 형태를 완벽하게 수치화하여 잡아냈습니다. 최종적으로 `cv2.initUndistortRectifyMap` 함수를 통해 이펙트로 인해 찌그러졌던 공간을 다시 평평하고 정상적인 직선 형태(Normality)로 역보정해 내는 데 성공했습니다.
